Clasificación por Modelo de Datos

Los sistemas de bases de datos se clasifican según el modelo de datos que utilizan para organizar y almacenar la información. Esta clasificación determina cómo se estructuran, almacenan y consultan los datos.

Bases de Datos Relacionales (RDBMS)

Sistemas basados en el modelo relacional que organizan datos en tablas con filas y columnas, garantizando integridad referencial mediante el álgebra relacional.

Bases de Datos NoSQL

Sistemas que no utilizan el modelo relacional, diseñados para manejar grandes volúmenes de datos no estructurados con mayor flexibilidad y escalabilidad horizontal.

Bases de Datos en Memoria

Sistemas que almacenan datos principalmente en memoria RAM para lograr máximo rendimiento, sacrificando persistencia por velocidad.

Teorema CAP (Consistency, Availability, Partition Tolerance)

Consistencia
Todos los nodos ven los mismos datos
Disponibilidad
Sistema responde a peticiones
Tolerancia a Particiones
Funciona ante fallos de red
Solo 2 de 3
Desafíos y Soluciones:

Consistencia

Desafío: Mantener datos sincronizados

Solución: Protocolos de consenso (Raft, Paxos)

Latencia de Red

Desafío: Comunicación entre nodos

Solución: Replicación asíncrona, CDN

Particionamiento

Desafío: División de datos

Solución: Sharding inteligente, rebalancing

-- PostgreSQL: Código abierto, extensible CREATE TABLE usuarios ( id SERIAL PRIMARY KEY, nombre VARCHAR(100) NOT NULL, email VARCHAR(255) UNIQUE, fecha_registro TIMESTAMP DEFAULT NOW() ); -- MySQL: Popular, fácil de usar CREATE TABLE usuarios ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(100) NOT NULL, email VARCHAR(255) UNIQUE, fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Oracle: Empresarial, alta disponibilidad CREATE TABLE usuarios ( id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, nombre VARCHAR2(100) NOT NULL, email VARCHAR2(255) UNIQUE, fecha_registro TIMESTAMP DEFAULT SYSTIMESTAMP ); -- SQL Server: Integración con Microsoft CREATE TABLE usuarios ( id INT IDENTITY(1,1) PRIMARY KEY, nombre NVARCHAR(100) NOT NULL, email NVARCHAR(255) UNIQUE, fecha_registro DATETIME2 DEFAULT GETDATE() );

Base de Datos en Memoria - Redis

Escenario: Sistema de cache y sesiones

import redis import json from datetime import timedelta # Configuración de Redis redis_client = redis.Redis( host='localhost', port=6379, db=0, decode_responses=True ) class CacheManager: def __init__(self, redis_client): self.redis = redis_client def cache_user_data(self, user_id, user_data, ttl=3600): """Cache de datos de usuario""" key = f"user:{user_id}" self.redis.setex(key, ttl, json.dumps(user_data)) def get_user_data(self, user_id): """Obtener datos de usuario del cache""" key = f"user:{user_id}" data = self.redis.get(key) return json.loads(data) if data else None def cache_product_list(self, category, products, ttl=1800): """Cache de lista de productos""" key = f"products:{category}" self.redis.setex(key, ttl, json.dumps(products)) def get_cached_products(self, category): """Obtener productos del cache""" key = f"products:{category}" data = self.redis.get(key) return json.loads(data) if data else None # Uso del cache manager cache = CacheManager(redis_client) # Cache de datos de usuario user_data = { "id": 123, "name": "John Doe", "email": "john@example.com", "preferences": {"theme": "dark"} } cache.cache_user_data(123, user_data)
